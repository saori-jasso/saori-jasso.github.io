OSI (Open System Interconnection)

Modelo conceptual que clasifica las funciones de comunicación en siete capas, las cuales son las siguientes: Física, Enlace de datos, Red, Transporte, Sesión, Presentación y Aplicación. Dicho modelo nos sirve para identificar problemas de red, entenderlas y ser capaces de diseñar sistema de redes.

IP (Internet Protocol)

Protocolo de la capa de red (mencionado en el modelo OSI). Se encarga de direccionar y enrutar paquetes de datos en redes. Gracias a este protocolo se identifican los dispositivos y se logran transmitir datos a través de redes interconectadas.

TCP (Transmission Control Protocol)

Protocolo que opera en la capa de transporte. Asegura la entrega ordenada, completa y sin errores de datos entre diferentes dispositivos mediante una conexión confiable basada en confirmaciones y retransmisiones.

UDP (User Datagram Protocol)

Protocolo que también opera en la capa de transporte. Es caracterizado por ser no confiable, pero más veloz que TCP, esto debido a que no verifica la entrega de paquetes. Es utilizado en algunos juegos online proporcionando eficiencia y velocidad.

SSH (Secure Shell)

Es un protocolo de red que posibilita la conexión segura a dispositivos a través de redes inseguras, utilizando encriptación para proteger la comunicación entre cliente y el servidor.

FTP (File Transfer Protocol)

Protocolo de red diseñado y utilizado específicamente para la transferencia de archivos entre un cliente y un servidor. Le permite a los usuarios subir, descargar o modificar archivos a través de servidores remotos.

SMTP (Simple Mail Transfer Protocol)

Protocolo que opera en la capa de aplicación del modelo OSI y se utiliza para enviar correos electrónicos entre servidores. Su función es facilitar la transmisión de mensajes, asegurando que estos lleguen al servidor del destino. Para recibir dichos correos se trabaja en conjunto de otros protocolos como POP.

POP (Post Office Protocol)

Es un protocolo de la capa de aplicación utilizado para la recepción de correos electrónicos desde un servidor de correo. A diferencia de SMTP el cual se encarga de enviar dichos mensajes de correo.

DHCP (Dynamic Host Configuration Protocol)

Protocolo de red que se encarga de la asignación dinámica de direcciones IP a los diferentes dispositivos dentro de una red. A través de DHCP, los dispositivos adquieren una dirección IP de manera rápida, lo que facilita la gestión de redes.

DNS (Domain Name System)

Es un sistema el cual traduce los nombres de dominio que nosotros leemos como “www.Instagram.com” en direcciones IP numéricas, las cuales ayudan a localizar y a acceder a los recursos del mismo internet. Gracias a este protocolo, se facilita la navegación web, permitiendo que los navegadores encuentren los recursos correspondientes.

HTTP (Hypertext Transfer Protocol)

Es un protocolo de la capa de aplicación utilizado para transmisión de documentos web en la World Wide Web. Este protocolo constituye un mecanismo esencial para la comunicación entre navegadores y servidores web. HTTP es el estándar y HTTPS es la variante segura.

ARP (Adress Resolution Protocol)

Es un protocolo el cual vincula una dirección IP, con una dirección MAC en una red de área local. Este proceso es fundamental, porque las direcciones IP y MAC son formatos distintos, por lo que este protocolo actúa como traductor, convirtiendo una dirección de 32 bits a una de 48 bits o viceversa.

MAC (Media Access Control)

Funcionan como identificadores únicos, compuestos por 48 bits. Es decir, son direcciones físicas exclusivas asignada a cada interfaz de red de un dispositivo, lo que permite identificarlo dentro de una una red local. Estas direcciones se presentan comúnmente en formato hexadecimal.

LAN (Local Area Network)

Es un conjunto de dispositivos electrónicos interconectados dentro de un área geográficamente limitada, como un domicilio, oficina, cafetería u otros espacios. Dichos dispositivos están configurados para comunicarse y compartir recursos entre sí.
